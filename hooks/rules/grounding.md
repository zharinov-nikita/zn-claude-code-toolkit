# Заземление: ctx7 + web search вместо памяти

**Не отвечай по памяти там, где можно проверить. Галлюцинированный API хуже честного «не знаю».**

## Инструменты

- **Документация библиотек** — Context7 CLI (установлен глобально), через Bash/PowerShell:
  1. `ctx7 library <имя> "<вопрос>"` — найти library ID;
  2. `ctx7 docs <libraryId> "<вопрос>"` — получить актуальные доки по вопросу.
- **Всё остальное** — `WebSearch` / `WebFetch`: версии, changelog, breaking changes, поведение сервисов, цены, новости, всё после knowledge cutoff.

## Когда проверять ОБЯЗАТЕЛЬНО

- Точные сигнатуры API, имена методов/опций внешних библиотек и фреймворков.
- Форматы конфигов (поля, ключи, дефолты).
- Выбор версии зависимости, совместимость версий.
- Всё, что могло измениться после knowledge cutoff, и всё, в чём есть хоть какая-то неуверенность.

## Когда можно без проверки

- Базовый синтаксис языка, стандартная библиотека.
- Общеизвестные стабильные вещи (git, POSIX, SQL-основы).
- Код, который уже есть в проекте пользователя (его читай напрямую).

## Правила

- Проверяй **до** написания кода, а не после того как он не заработал.
- Если ctx7 и web search недоступны или ничего не дали — так и скажи: «не проверял, пишу по памяти», чтобы пользователь знал степень надёжности.
- Не выдумывай library ID — сначала `ctx7 library`, потом `ctx7 docs`.
